<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| App -> config.json file | App writes and reads its own settings file; another process could tamper with or corrupt it |
| App -> HKCU Run registry key | App writes a launch path that the OS executes at login |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Denial of Service | load_config() reading malformed config.json | mitigate | Catch (json.JSONDecodeError, OSError) and fall back to _DEFAULTS — verified by malformed-file test |
| T-04-02 | Tampering | config.json written with unexpected/extra keys by another process | mitigate | Merge loaded data over _DEFAULTS so missing keys are restored and unknown keys are preserved without crashing |
| T-04-03 | Tampering | Run key value pointing to wrong path (space-splitting) | mitigate | Quote sys.executable in outer double-quotes; path is never user-supplied input |
| T-04-04 | Elevation of Privilege | Writing startup entry without admin | accept | HKCU (current user) requires no admin by design; HKLM is never touched |
| T-04-SC | Tampering | pip install of pytest-qt (supply chain) | mitigate | slopcheck + blocking-human checkpoint (Task 1) verifies pytest-dev origin before install |
</threat_model>
