<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Background asyncio thread -> Qt main thread | DeviceState crosses via queue.Queue drained by QTimer; only the drain touches Qt widgets |
| User -> Settings toggle -> registry/file | UI action triggers a winreg + file write through SettingsManager |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-05 | Tampering | Cross-thread widget access from bg thread | mitigate | All widget mutation happens only inside on_device_update, invoked by the main-thread QTimer drain — bg thread only calls queue.put (architecture invariant, unchanged) |
| T-04-06 | Denial of Service | Missing tray_icon.png at runtime | mitigate | Generate the asset in Task 2 and resolve its path relative to __file__ so cwd cannot break it |
| T-04-07 | Elevation of Privilege | COM init order regression in new __main__.py | mitigate | sys.coinit_flags=0 kept as statement 2 before all non-sys imports; verified by AST check in Task 3 |
| T-04-08 | Denial of Service | App fails to quit (window hidden, no tray) leaving orphan process | mitigate | setQuitOnLastWindowClosed(False) + explicit Quit action calling qapp.quit(); verified by tray test |
</threat_model>
