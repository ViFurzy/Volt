<!-- Hardware-confirmed protocol facts from 02-01-SUMMARY.md -->

G Pro X Wireless HID interfaces discovered:
  usage_page=0xFF00 — 64-byte output, report IDs 0x00/0x90/0xC4 — WRONG INTERFACE (writes return -1)
  usage_page=0xFF43 — 20-byte output, report ID 0x11 — CORRECT INTERFACE

Fixed device index: 0xFF (the LIGHTSPEED receiver itself; G-series headset protocol does not use
per-device index probing via Root feature 0x0000)

From src/hid_poc.py (open_path pattern to reuse verbatim):
```python
LOGITECH_VID      = 0x046D
VENDOR_USAGE_PAGE = 0xFF00   # NOTE: hid_poc.py uses 0xFF00; receiver.py uses 0xFF43

all_devices = hid.enumerate(vid, 0)
vendor_interfaces = [d for d in all_devices if d["usage_page"] == VENDOR_USAGE_PAGE]

device = hid.device()
try:
    device.open_path(info["path"])
finally:
    device.close()
```

bytes-path guard (copy from hid_poc.py — prevents UnicodeDecodeError on malformed OS paths):
  path = info["path"]
  if isinstance(path, bytes):
      path = path.decode("utf-8", errors="replace")
