  <action>
  Create src\hid_poc.py as a standalone script (no Qt, no asyncio — only the hid library).

  The script must execute these steps in order when run as __main__:

  STEP 1 — Full Logitech VID scan (required — see Open Question #1 in RESEARCH.md):
    Call hid.enumerate(0x046D, 0) to enumerate ALL devices with VID=0x046D (passing 0 as PID means all PIDs).
    Print every result: PID (hex), usage_page (hex), usage (hex), path (decoded as utf-8 or repr if bytes), manufacturer_string, product_string.
    This output is the hardware discovery step — it reveals the exact PID physically present, which may differ from the expected 0xC547.
    Label the output: "=== All Logitech (0x046D) HID interfaces ==="

  STEP 2 — Filter for LIGHTSPEED candidates:
    From the enumeration in Step 1, collect all entries where usage_page == 0xFF00.
    If none found: print a clear error message listing all PIDs seen, and exit with sys.exit(1). Do not attempt to open anything.
    Print the count: "Found N interface(s) with usage_page=0xFF00"
    Use the FIRST such entry (index 0) for the open attempt.

  STEP 3 — Open via open_path():
    Create a hid.device() instance.
    Call device.open_path(info["path"]) where info is the first 0xFF00 entry from Step 2.
    NEVER use hid.device().open(vid, pid) — this opens the wrong interface (see RESEARCH.md Pitfall 1 and Architecture Invariant in CLAUDE.md).
    After opening, print:
      "Opened: <manufacturer_string> <product_string>"
      "  VID: 0x<vid:04X>  PID: 0x<pid:04X>  usage_page: 0xFF00"

  STEP 4 — Raw write + read round-trip:
    Write a 64-byte null output report:
      payload = [0x00] * 64  (report ID 0x00 = no report ID prefix; the device will ignore it)
    Call device.write(payload).
    Print: "Write: sent 64-byte null report"
    Read back with timeout:
      response = device.read(64, timeout_ms=1000)
    Wrap BOTH write and read in try/except that catches OSError and Exception separately, printing the exception type and message.
    After read: check with "if not response:" (empty list = timeout). Print:
      - On timeout: "Read: timeout (empty response) — device may be off or wrong interface"
      - On data: "Read: {len(response)} bytes — first 8: {response[:8]}"
    This satisfies Success Criterion 2 (bytes out, bytes back). An empty read is acceptable — it proves the interface opened correctly.

  STEP 5 — Close:
    Call device.close() in a finally block that wraps Steps 3 and 4.

  Script structure:
    - Top of file: a module-level docstring explaining the script's purpose (one paragraph)
    - Standalone functions: find_vendor_interfaces(vid), open_and_probe(info)
    - A main() function that calls them in order
    - if __name__ == "__main__": guard calling main()
    - NO sys.coinit_flags line — this script is not __main__; that guard lives only in src/__main__.py

  Exact constants to define at module level:
    LOGITECH_VID = 0x046D
    VENDOR_USAGE_PAGE = 0xFF00
    REPORT_SIZE = 64
    READ_TIMEOUT_MS = 1000
  </action>

|----------|-------------|
| Python process -> Windows HID driver | Raw bytes written to HID output report; response bytes read from HID input report |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | hid_poc.py response buffer | mitigate | Read fixed 64 bytes; slice response[:8] for logging; never interpret raw bytes as executable content in Phase 1 |
| T-01-02 | Tampering | hid.enumerate() path field | accept | Path is a bytes object returned by hidapi; no user input involved; low-value target in PoC context |
| T-01-SC | Tampering | pip install hid==1.0.9 PySide6==6.11.1 | mitigate | Package Legitimacy Audit in RESEARCH.md confirms both packages; exact version pins prevent substitution attacks |
</threat_model>
