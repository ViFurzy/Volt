  <files>src/monitor/registry.py, tests/test_registry.py</files>
  <read_first>
    - src/monitor/state.py (DeviceState, DeviceStatus — created in Task 1)

  </read_first>
  <behavior>
    - upsert(state) then get((vid, pid, dev_idx)) returns the same DeviceState
    - upsert with the same key twice overwrites (latest snapshot wins, per D-02)
    - get on an unknown key returns None
    - all() returns a list/snapshot copy of all stored DeviceState values
    - mark_offline(key) sets the stored DeviceState.status to DeviceStatus.OFFLINE and percent to None, returning the updated state (None if key absent)
    - concurrent upserts from multiple threads do not corrupt the store (no lost writes, no exceptions)
  </behavior>
  <action>
    Create `src/monitor/registry.py` with a `class DeviceRegistry`:
    - `__init__` creates `self._lock = threading.Lock()` and `self._devices: dict[tuple[int, int, int], DeviceState] = {}`.
      Per Claude's Discretion in CONTEXT: a `threading.Lock` around a plain dict (simplest model
      matching threading_stub.py).
    - Key is the 3-tuple `(state.vid, state.pid, state.dev_idx)`.
    - `upsert(self, state: DeviceState) -> None`: acquire lock, store `state` under its key (overwrite).
    - `get(self, key: tuple[int, int, int]) -> DeviceState | None`: acquire lock, return `self._devices.get(key)`.
    - `all(self) -> list[DeviceState]`: acquire lock, return `list(self._devices.values())` (copy, not live view).
    - `mark_offline(self, key: tuple[int, int, int]) -> DeviceState | None`: acquire lock; if key present,
      replace stored state with a copy having `status=DeviceStatus.OFFLINE` and `percent=None`
      (use `dataclasses.replace`), store it, return it; else return None. This is the HID-04 teardown
      helper consumed by the hot-plug unplug path in 03-03.
    Use `with self._lock:` for every method body. Do NOT use asyncio locks — this object is touched
    from both the asyncio bg thread and (read-only) the main thread.

    Append DeviceRegistry tests to `tests/test_registry.py` covering the <behavior> list, including a
    concurrency test that spawns several threads each calling upsert in a loop and asserts no exception
    and final count is correct.
  </action>

| Boundary | Description |
|----------|-------------|
| bg asyncio thread ↔ main Qt thread | DeviceRegistry is the shared mutable store touched from both threads; the lock is the boundary guard |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | DeviceRegistry dict shared across threads | mitigate | Every read/write body wrapped in `with self._lock:`; `all()` returns a copy not a live view, preventing iteration-during-mutation corruption |
| T-03-02 | Information Disclosure | KNOWN_DEVICES / DeviceState | accept | No PII or secrets — only VID/PID and battery percent; local-only app, no network |
| T-03-SC | Tampering | npm/pip installs | accept | No new dependencies installed in this plan (stdlib enum, dataclasses, threading only) |
</threat_model>
