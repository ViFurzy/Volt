<objective>
Build `MonitorService` — the asyncio background-thread service that owns the 60-second polling
loop, discovers known devices, reads battery via the Phase 2 protocol layer, and pushes full
`DeviceState` snapshots to a thread-safe `queue.Queue`. This is the core engine of Phase 3.
Hot-plug reception (Qt main-thread WM_DEVICECHANGE) is added in 03-03; this plan builds the
asyncio side that hot-plug will call into via `run_coroutine_threadsafe`.

Purpose: Satisfy the polling/discovery half of HID-03 (auto-discover known devices) and the
mid-session detection half of HID-04 (a device that stops responding is marked OFFLINE). The
loop runs on the canonical daemon asyncio thread from threading_stub.py without blocking the
main thread.
Output: `src/monitor/service.py`, a `verbose` flag added to `find_receiver`, and
`tests/test_service.py`.
</objective>

  <action>
    Create `src/monitor/service.py` with a `class MonitorService`. Follow the threading_stub.py model
    exactly (asyncio loop runs on a daemon thread via run_forever; clean shutdown via
    loop.call_soon_threadsafe(loop.stop)).

    Constructor `__init__(self, ui_queue: queue.Queue, registry: DeviceRegistry, poll_interval: float = 60.0)`:
    store the queue, the registry, the interval; create `self._loop = asyncio.new_event_loop()`;
    `self._thread: threading.Thread | None = None`; `self._open: dict[tuple[int,int,int], hid.device] = {}`
    to track open handles per device key; `self._poll_task = None`.

    Methods:
    - `start(self) -> None`: launch the daemon thread running `_run_loop` (set_event_loop + run_forever),
      then `asyncio.run_coroutine_threadsafe(self._poll_loop(), self._loop)`. Mirror threading_stub
      run_background.
    - `_run_loop(self) -> None`: `asyncio.set_event_loop(self._loop); self._loop.run_forever()`.
    - `async def _poll_loop(self)`: on first entry call `await self.discover()`; then loop forever:
      `await self.poll_once()` then `await asyncio.sleep(self.poll_interval)`.
    - `async def discover(self) -> None`: call `find_receiver(verbose=False)`; for each returned
      interface dict, build key from `(info["vendor_id"], info["product_id"], DEVICE_IDX)`; skip any
      whose `(vendor_id, product_id)` is NOT in KNOWN_DEVICES (HID-03: only known devices). For each
      kept device: if not already in `self._open`, `open_receiver(info)` and store the handle; look up
      `device_name = KNOWN_DEVICES[(vid, pid)]` (per D-05, not product_string); upsert an initial
      `DeviceState(..., percent=None, charging=False, status=DeviceStatus.ONLINE)` into the registry
      and put it on the queue. Wrap open in try/except OSError; on failure skip that device.
    - `async def poll_once(self) -> None`: for each open device key+handle, call
      `battery_probe_chain(handle, DEVICE_IDX)`. If result is None (device off/offline mid-session):
      call `registry.mark_offline(key)`, push that OFFLINE DeviceState to the queue, close the handle,
      and remove it from `self._open` (per CONTEXT discretion: do NOT keep hammering a closed handle;
      wait for next WM_DEVICECHANGE plug to re-open). If result is a BatteryResult: map charging→status
      via the D-03 rule (charging True → DeviceStatus.CHARGING; else DeviceStatus.ONLINE), build a full
      DeviceState snapshot, `registry.upsert(state)`, and `ui_queue.put(state)` (full snapshot per D-02).
    - `def rescan(self) -> concurrent.futures.Future`: thread-safe entry point for the hot-plug callback
      in 03-03 — `return asyncio.run_coroutine_threadsafe(self.discover(), self._loop)`. (03-03 wraps this
      with debounce.)
    - `def stop(self) -> None`: close every handle in self._open; `self._loop.call_soon_threadsafe(self._loop.stop)`;
      `self._thread.join(timeout=5.0)` if thread alive (mirror threading_stub shutdown). Guard for None thread.

    Use `import hid` only for the type annotation comment; the service never calls hid.open directly —
    it goes through receiver.open_receiver (invariant: open by path/usage_page, never hid.open(vid,pid)).
    All HID calls happen inside coroutines running on self._loop (bg thread). The service NEVER touches Qt.

    Create `tests/test_service.py`: use the `mock_hid` conftest fixture and `mocker` to patch
    `monitor.service.find_receiver`, `monitor.service.open_receiver`, and
    `monitor.service.battery_probe_chain`. Test discover/poll_once as plain awaited coroutines via
    `asyncio.run(...)` (no real thread/hardware). Cover the <acceptance_criteria> mapping cases.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HID device → MonitorService | Raw bytes from the dongle parsed by battery_probe_chain; malformed/short responses already return None in the protocol layer |
| bg asyncio loop ↔ main thread | Hot-plug callback (03-03) crosses here via run_coroutine_threadsafe(rescan); queue.put crosses back |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-03 | Denial of Service | poll_once on a dead handle | mitigate | On None result, close handle and remove from self._open — never re-poll a closed handle (CONTEXT discretion); re-open only on hot-plug |
| T-03-04 | Tampering | open device handle reused across threads | mitigate | All HID I/O confined to coroutines on self._loop (bg thread) per CLAUDE.md invariant; rescan() is the only thread-safe entry and only schedules onto the loop |
| T-03-05 | Spoofing | unknown VID/PID device enumerated | mitigate | discover() filters every interface against KNOWN_DEVICES; non-listed devices are never opened or registered (HID-03) |
| T-03-SC | Tampering | npm/pip installs | accept | No new dependencies — stdlib asyncio/threading/queue plus existing hid + hidpp modules only |
</threat_model>
