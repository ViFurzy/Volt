<objective>
Wire all Phase 3 components into a single runnable application and prove the end-to-end queue pipe:
`MonitorService` (bg polling) + `DeviceRegistry` (shared store) + `HotPlugWatcher` (hot-plug) feed
`DeviceState` snapshots through `queue.Queue` to a QTimer-driven drain on the Qt main thread that
hands them to a mock consumer. A standalone entry point (`run_monitor.py`) with the
`sys.coinit_flags = 0` guard runs the whole stack and prints received DeviceStates, and a hardware
checkpoint confirms plug/unplug behavior on the real G Pro X Wireless dongle.

Purpose: Satisfy Phase 3 success criterion 5 (mock consumer on the main thread receives DeviceState
updates end-to-end) and provide the integration seam Phase 4 replaces the mock consumer with the
real device-card UI. Closes HID-03 (auto-discover on plug) and HID-04 (offline on unplug) at the
system level.
Output: `src/monitor/app.py`, `src/run_monitor.py`, `tests/test_integration.py`.

    (QTimer) because it runs on the main thread; it must never perform HID I/O.
