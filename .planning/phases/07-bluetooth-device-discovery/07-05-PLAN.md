  <action>
    Based on the human checkpoint report, fix any issues found.

    Common fixes to expect:

    Issue: "Scan button stuck on Scanning...":
    - The done callback from run_coroutine_threadsafe runs on bg thread; UI update must happen via the queue drain, not the callback.
    - Fix: on_scan_result() is called by drain() on the Qt main thread — ensure the button re-enable is inside on_scan_result(), not in a done_callback.

    Issue: "Dashboard card not appearing after Add":
    - The card is created by on_bt_device_update() which is triggered by BtDeviceInfo from poll_once().
    - Fix: check that poll_once() BT loop correctly uses the newly-added device from self._bt_devices, and that drain() routes BtDeviceInfo to bt_consumer.

    Issue: "App crashes on startup after persistence test":
    - Load error in monitored_devices config entry (missing keys).
    - Fix: add .get() defaults in discover() BT loading block.

    Issue: "Devices page scan results empty when BT devices are paired":
    - winrt_enumerate_bt() may be returning [] due to an import error or WinRT exception swallowed by the outer try/except.
    - Fix: add logging/print to winrt_enumerate_bt() to surface any exception.

    After any fix, re-run:
      python -m pytest tests/ -x -q

    If no issues were reported, this task is a no-op — write SUMMARY and proceed.
