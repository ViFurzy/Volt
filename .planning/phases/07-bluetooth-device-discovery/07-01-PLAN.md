    async def winrt_enumerate_bt() -> list[dict]:
    - Import inside function body: from winrt.windows.devices.enumeration import DeviceInformation; from winrt.windows.devices.bluetooth import BluetoothDevice
    - Call BluetoothDevice.get_device_selector_from_pairing_state(True) for AQS
    - additional_props = ["System.ItemNameDisplay", BATTERY_PKEY]
    - devices = await DeviceInformation.find_all_async(aqs, additional_props)
    - For each device d: battery_raw = d.properties.get(BATTERY_PKEY); battery_pct = int(battery_raw) if isinstance(battery_raw, (int, float)) else None
    - Return list of dicts with keys: id (d.id), name (d.name), battery (battery_pct), type ("bt")
    - Wrap entire body in try/except Exception returning [] on any WinRT error

    async def gatt_battery(ble_address: str, timeout: float = 5.0) -> int | None:
    - Use BleakClient as async context manager with timeout parameter
    - await client.read_gatt_char(BATTERY_CHAR_UUID) → bytes
    - Return int.from_bytes(data[:1], "little") if data else None
    - Catch BleakCharacteristicNotFoundError → return None
    - Catch BleakError → return None
    - Catch Exception → return None (covers asyncio.TimeoutError etc.)

    async def resolve_battery(device_info: dict) -> int | None:
    - Tier (a): winrt_pct = device_info.get("battery"); if winrt_pct is not None: return winrt_pct
    - Tier (b): ble_address = device_info.get("ble_address"); if ble_address: gatt_pct = await gatt_battery(ble_address); if gatt_pct is not None: return gatt_pct
    - Tier (c): return None (HID++ is handled separately by MonitorService for KNOWN_DEVICES)

    Imports: asyncio at top level; bleak imports inside gatt_battery function body to allow module load even if bleak is not installed.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| WinRT property bag → bt_backend | d.properties.get(BATTERY_PKEY) returns arbitrary Python object from OS; must type-check before int() |
| BLE GATT read → bt_backend | bytes from remote device; must length-check before int.from_bytes() |
| Paired BT device identity | Only paired (trusted) devices are enumerated; rogue device spoofing is out of scope |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | Tampering | winrt_enumerate_bt property bag | mitigate | isinstance(battery_raw, (int, float)) guard before int() conversion; catches str, None, unexpected type |
| T-07-02 | Tampering | gatt_battery byte decode | mitigate | data[:1] slice + length check (if data) before int.from_bytes(); catches empty bytes |
| T-07-03 | Spoofing | BleakClient address | accept | BleakClient takes device address (not name); address spoofing at OS layer is outside app scope |
| T-07-SC | Tampering | pip install bleak, winrt-* | mitigate | Package legitimacy audit in RESEARCH.md; bleak [SUS] overridden as documented false positive; all [OK] packages from official pywinrt |
</threat_model>
